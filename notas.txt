Nesta branch, queremos ir além e fazer com que o ESLint reporte erros de formatação de acordo com as regras do Prettier.
_ Instale o pacote: npm install --save-dev eslint-plugin-prettier
- No arquivo .eslintrc.json:
  ✔️ "plugins": ["prettier"],
  ✔️ "extends": ["eslint:recommended","airbnb","prettier"],
  ✔️ "rules": {
    "prettier/prettier": "error",
    "arrow-body-style": "off",
    "prefer-arrow-callback": "off"
  }
- Altere o arquivo .prettierrc para:
  ✔️ {
      "singleQuote": true
    }
- Recarregue o VS Code, caso necessário
- No arquivo main.js, observe que:
✔️ o ESLint reporta erro pelo uso de aspas duplas, pois Prettier, agora, requer aspas simples
✔️ Passe o mouse sobre a string "result" e perceba que o erro vem da regra do plugin: 
  ✔️ Replace `"result"` with `'result'`eslintprettier/prettier
✔️ Salve o arquivo e Prettier formata e corrige o problema e que ESLint para de reportar o erro, pois já não existe
- Podemos simplificar no .eslintrc.json:
  ✔️ Remova a propriedade "plugins"
  ✔️ "extends": ["eslint:recommended", "airbnb", "plugin:prettier/recommended"],
  ✔️ Remova a propriedade "rules"
- Recarregue p VS Code e observe que tudo funciona como antes
- Altere o arquivo .prettierrc:
  ✔️ {
      "printWidth": 100,
      "semi": false,
      "singleQuote": true,
      "tabWidth": 2,
      "useTabs": false,
      "trailingComma": "all"

- Abra o main.js e salve para aplicar a nova formatação do Prettier
- Corrija o erro reportado na arrow function
- Deixe apenas um console.log
- Corrija o nome da variavel no console.log

    }
