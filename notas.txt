- Instale o plugin Prettier - Code formatter no VS Code
- Instale o Prettier com dependência de denvolvimento no projeto:
  ✔ npm install prettier -D --save-exact
- Configure o Prettier para uso no VS Code
  ✔ (CTRL+SHIFT+P) >settings (opção JSON)
  ✔ Adicione as linhas 
    ✔ "editor.formatOnSave": true,
    ✔ "editor.defaultFormatter": "esbenp.prettier-vscode"
    ✔ "prettier.requireConfig": true,
    ✔ "prettier.resolveGlobalModules": false,
  ✔ Crie um arquivo de configuração .prettierrc
  ✔ Adicione um objeto de configuração:
    ✔ {
        "singleQuote": true
      }
  ✔️ No arquivo main.js triplique a linha de console, como abaixo:
    ✔️ console.log("result", resultado);
      console.log("result", resultado);
      console.log(`result`, resultado);    
  ✔️ Salve o arquivo e observe que a formatação foi substituída para single quotes, exceto a que usa backtick
  ✔️ No arquivo .eslintrc.json adicione a seguinte rule:
    ✔ "rules": {
        "quotes": ["error", "double", { "allowTemplateLiterals": true }]
      }

‼ Observe que agora há conflito entre o que ESLint exige e o que Prettier faz.
Neste casp específico seria fácil resolver, pois a regra "quotes" não é uma regra "recommended",
nós que a incluímos e configuramos,
mas há casos em que regras "recommended" entram em conflito com regras de formatação do Prettier.
