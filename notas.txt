- Vamos delegar ao Prettier a responsibilidade de formatar nosso código, 
  desabilitando todas as regras que possam entrar em conflito,
  mas antes vamos instalar e configurar outro conjunto de regras bastante conhecido:
  ✔️ npm install --save-dev eslint-config-airbnb
  ✔️ "extends": ["eslint:recommended", "airbnb"],
  ✔️ Comente as regras:
    "rules": {
      //"quotes": ["error", "double", { "allowTemplateLiterals": true }]
    }
- Observe que agora o ESLint reporta outros problemas no arquivo main.js
  ❗ um erro na arrow function
  ❕ um warning na chamada ao console
 - Altere o arquivo .prettierrc para:
    {
      "singleQuote": false
    }
- Abra e salve o arquivo main.js
- Observe que as aspas mudaram para duplas e o ESLint reclamou, pois a configuração do airbnb requer aspas simples
- Rode o comando: npm run lint-fix
- Observe que agora as aspas voltam a ser simples, mas se salvamos o arquivo o Prettier as converte para duplas novamente.
  🤯 🤯 🤯 
- Vamos instalar outro pacote:
  ✔️ npm install --save-dev eslint-config-prettier
- No arquivo .eslintrc.json:
  ✔️ "extends": ["eslint:recommended", "airbnb", "prettier"],
  ✔️ Recarregue o VS Code para carregar as novas configurações, caso necessário
- Agora se salvamos o arquivo, o Prettier entra em ação com as aspas duplas e o ESLint não reclama
